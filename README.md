# 🔐 Менеджер паролей для терминала

Консольное приложение для безопасного хранения паролей с шифрованием и удобным интерфейсом.

## 📋 Что это такое?

Это программа для компьютера, которая помогает хранить все ваши пароли в одном безопасном месте. Представьте себе сейф, куда вы кладёте все свои пароли от социальных сетей, банков, почты и других сайтов. Только вы можете открыть этот сейф с помощью особого ключа — **мастер-пароля**.

### Основные возможности:

- 🔒 **Шифрование** — все пароли хранятся в зашифрованном виде
- 🖥️ **Удобный интерфейс** — работает прямо в терминале/командной строке
- 📋 **Автокопирование** — пароли копируются в буфер обмена одним нажатием
- ⚡ **Генератор паролей** — создаёт надёжные пароли автоматически
- 🌐 **Русский язык** — интерфейс полностью на русском

Программа построена по принципу **разделения ответственности** — каждая часть отвечает за свою задачу:

```
┌─────────────────────────────────────────────────────────────┐
│                    Пользователь                             │
│                 (вы в терминале)                            │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  UI (User Interface) — отвечает за отображение интерфейса   │
│  Файлы: src/ui/components.ts, src/ui/password-manager-ui.ts │
│  • Рисует меню, формы, таблицы                              │
│  • Обрабатывает нажатия клавиш                              │
│  • Показывает сообщения                                     │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  App (Application) — связывает UI и бизнес-логику           │
│  Файл: src/app.ts                                           │
│  • Решает, что делать при выборе пункта меню                │
│  • Передаёт данные между UI и сервисами                     │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Services (Сервисы) — бизнес-логика и данные                │
│  Файл: src/services/encrypted-password-service.ts           │
│  • Шифрует и расшифровывает пароли                          │
│  • Сохраняет данные в файл                                  │
│  • Проверяет мастер-пароль                                  │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Utils (Утилиты) — вспомогательные функции                  │
│  Файл: src/utils/password-generator.ts                      │
│  • Генерирует случайные пароли                              │
└─────────────────────────────────────────────────────────────┘
```

### Поток данных при сохранении пароля:

1. Вы вводите данные в форму (UI)
2. App получает эти данные и передаёт в сервис
3. Сервис шифрует пароль и сохраняет в файл
4. UI показывает сообщение об успехе

### Детальная установка для Windows

Если у вас возникли проблемы, вот подробная инструкция:

#### Шаг 1: Установка Bun на Windows

**Способ 1 — Через PowerShell (рекомендуется):**

1. Нажмите `Win + X`, выберите **Windows PowerShell (администратор)**
2. Вставьте команду:
   ```powershell
   powershell -c "irm bun.sh/install.ps1 | iex"
   ```
3. Нажмите Enter и дождитесь окончания установки
4. **Закройте PowerShell полностью** и откройте заново

**Способ 2 — Через npm (если уже установлен Node.js):**

```powershell
npm install -g bun
```

**Способ 3 — Скачивание вручную:**

1. Перейдите на https://github.com/oven-sh/bun/releases
2. Скачайте файл `bun-windows-x64.zip`
3. Распакуйте в папку `C:\bun`
4. Добавьте `C:\bun` в переменную среды PATH:
   - Нажмите `Win + R`, введите `sysdm.cpl`, нажмите Enter
   - Вкладка **Дополнительно** → **Переменные среды**
   - В разделе **Системные переменные** найдите `Path` → **Изменить**
   - **Создать** → введите `C:\bun` → **OK**

#### Шаг 2: Проверка установки

```powershell
bun --version
# Должно показать версию, например: 1.1.0
```

#### Шаг 3: Установка проекта

```powershell
# Скачайте проект (или распакуйте ZIP-архив)
cd C:\Users\ВашеИмя\Documents
git clone https://github.com/username/password-manager.git

# Или если скачали ZIP:
# Expand-Archive -Path password-manager.zip -DestinationPath .

# Перейдите в папку
cd password-manager

# Установите зависимости
bun install
```

#### Шаг 4: Запуск

```powershell
bun run index.ts
```

## 🚀 Использование

### Первый запуск

При первом запуске программа создаст новое хранилище паролей:

1. Запустите программу: `bun run index.ts`
2. Введите **мастер-пароль** — это ваш главный ключ для доступа ко всем паролям
   - Придумайте сложный пароль (минимум 8 символов)
   - Запомните его! Без него невозможно восстановить данные
3. Подтвердите вход

### Основные действия

| Клавиша | Действие |
|---------|----------|
| `↑` `↓` или `Ctrl+N` `Ctrl+P` | Навигация по меню |
| `Enter` | Выбрать пункт меню |
| `Tab` | Переход между полями в форме |
| `Ctrl+G` | Сгенерировать пароль автоматически |
| `q` | Выход из программы |
| `Esc` | Отмена / Вернуться назад |

### Где хранятся пароли?

Все пароли сохраняются в файл `passwords.dat` в папке проекта. Этот файл зашифрован — без мастер-пароля его невозможно прочитать.

Вы можете перенести этот файл на другой компьютер — пароли будут доступны при вводе того же мастер-пароля.

## 🔐 Безопасность

### Как работает шифрование?

```
┌─────────────────────────────────────────────────────────────┐
│  Ваш пароль от сайта: "MySecret123!"                        │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Ключ шифрования (256 бит)                                  │
│  Получается из вашего мастер-пароля                         │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Алгоритм AES-256-GCM                                       │
│  Превращает пароль в нечитаемый набор символов             │
└───────────────────────┬─────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│  Сохраняется в файл: "a7f3b2c9d8e1..." (зашифровано)       │
└─────────────────────────────────────────────────────────────┘
```

### Что это значит простыми словами:

- **AES-256-GCM** — это один из самых надёжных алгоритмов шифрования в мире
- Без мастер-пароля файл с паролями — просто бессмысленный набор букв и цифр
- Даже если кто-то получит файл `passwords.dat`, он не сможет прочитать ваши пароли

### Важные правила безопасности:

1. **Храните мастер-пароль в надёжном месте** — без него данные невосстановимы
2. **Не передавайте мастер-пароль никому**
3. **Делайте резервные копии** файла `passwords.dat`

## 🐛 Решение проблем

### Ошибка: "bun не найден"

**Windows:**
```powershell
# Проверьте, что Bun добавлен в PATH
$env:Path -split ";" | Select-String "bun"

# Если нет, добавьте вручную:
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Users\%USERNAME%\.bun\bin", "User")
```

**macOS/Linux:**
```bash
# Добавьте в ~/.bashrc или ~/.zshrc:
export PATH="$HOME/.bun/bin:$PATH"
```

### Ошибка: "Cannot find module"

```bash
# Удалите node_modules и установите заново:
rm -rf node_modules
bun install
```

### Интерфейс отображается некорректно

1. Убедитесь, что терминал поддерживает Unicode
2. Попробуйте другой терминал (Windows Terminal, iTerm2, GNOME Terminal)
3. Установите шрифт с поддержкой символов (например, Fira Code)
